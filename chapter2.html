Why Layered Architecture? — The Big Picture

لماذا نستخدم الطبقات؟ — الصورة الكاملة
📬 The Postal Mail Analogy تشبيه البريد

Networks require a combination of hardware and software. To make communications efficient, the problem is divided into layers — each layer with a specific function or service.

Think of sending a letter: you write it (cognitive layer), use a common language (linguistic layer), and the post office physically moves it (physical layer). Each "layer" does its own job without knowing the details of the others.

Arabic Explanation
لماذا نستخدم الطبقات؟ لأن التواصل الشبكي معقد جداً إذا حاولنا حله دفعة واحدة.
مثال البريد: المرسل يكتب الرسالة (طبقة المحتوى) → يكتب عليها عنواناً (طبقة العنونة) → يضعها في ظرف (طبقة التغليف) → يرسلها بالبريد (طبقة النقل الفيزيائي).
كل طبقة تؤدي عملها المحدد دون أن تعرف تفاصيل الطبقات الأخرى — هذا يُبسّط التصميم والتطوير والصيانة.
مثال التواصل بين أشخاص:
• Cognitive Layer: يحدد هدف الرسالة (ما أريد قوله)
• Linguistic Layer: يوفر لغة مشتركة (عربي، إنجليزي...)
• Physical Layer: ينقل المعلومات فيزيائياً (صوت، ورق، إلكترونيات)
🔑 Key Properties of Layered Architecture خصائص البنية الطبقية
  • Each layer defines distinct functions
  • Each layer calls services of the layer below it, and provides services to the layer above it
  • Between two machines, corresponding layers communicate with each other — called peer-to-peer processes
  • This inter-layer communication is governed by protocols
  • At the sender: each layer adds a header (encapsulation) going down
  • At the receiver: each layer removes its header (decapsulation) going up
Peer-to-Peer
The network layer on Device A "talks" logically to the network layer on Device B — even though physically the data passes through all layers at both ends.
Arabic
Peer-to-Peer: كل طبقة في الجهاز المُرسِل تتحدث "منطقياً" مع نظيرتها في الجهاز المستقبِل.
مثلاً: طبقة الشبكة في الكمبيوتر A تتواصل منطقياً مع طبقة الشبكة في الكمبيوتر B.
لكن فيزيائياً: البيانات تنزل من طبقة لطبقة في الجهاز المُرسِل، ثم تصعد في الجهاز المستقبِل.

عند الإرسال: كل طبقة تُضيف Header (معلومات التحكم الخاصة بها) → التغليف Encapsulation
عند الاستقبال: كل طبقة تنزع الـ Header الخاص بها → فك التغليف Decapsulation

The TCP/IP Internet Model — 5 Layers

نموذج الإنترنت TCP/IP — الخماسي
🌐 What is TCP/IP? ما هو نموذج TCP/IP؟

The dominant model in data communications and networking today. Developed from the ARPANET project. Has 5 ordered layers, often referred to as the TCP/IP protocol suite.

  • Developed from ARPANET (US Defense project)
  • Used by the entire modern Internet
  • 5 layers: Physical → Data Link → Network → Transport → Application
  • Real-world, practical model (not theoretical)
vs OSI
OSI is a theoretical 7-layer model. TCP/IP is the practical 5-layer model actually used. OSI adds Session and Presentation layers between Transport and Application.
Arabic
TCP/IP هو النموذج العملي المستخدم في الإنترنت الحقيقي اليوم.
طُوِّر من مشروع ARPANET الأمريكي العسكري — وأصبح معيار الإنترنت العالمي.
5 طبقات مرتبة من الأسفل للأعلى: فيزيائية → ربط بيانات → شبكة → نقل → تطبيق
الفرق عن OSI: TCP/IP عملي وأبسط (5 طبقات) | OSI نظري ومرجعي (7 طبقات)

Layer-by-Layer Deep Dive

تفصيل كل طبقة بعمق
🔌 Layer 1 — Physical Layer الطبقة الفيزيائية
Core Definition
The physical layer is responsible for transmitting individual bits from one node to the next.

Responsibilities:

  • Physical characteristics of interfaces and media (cable type, voltage levels, pin layout)
  • Bit representation — transmits raw bits without interpretation (no meaning applied)
  • Data rate — defines how many bits per second can be transmitted
  • Synchronization — sender and receiver clocks must agree when each bit starts/ends
  • Connection control — manages start and end of physical connection

Examples of Physical Media:

Copper Wire
Twisted pair, coaxial cable
Fiber Optic
Light pulses, very high speed
Wireless
Radio waves, WiFi, Bluetooth
Data Unit
Individual Bit (0 or 1)
Arabic — الطبقة الفيزيائية
الطبقة الفيزيائية مسؤولة عن نقل البتات الفردية (0 و1) من node لـ node.
لا تهتم بالمعنى — فقط تُرسِل الإشارات الكهربائية أو الضوئية أو الراديوية.
مسؤولياتها الخمس:
  ① الخصائص الفيزيائية: نوع الكابل، الجهد، التوصيلات
  ② تمثيل البتات: كيف يمثل الـ 0 والـ 1 (جهد عالي/منخفض)
  ③ معدل البيانات: كم بت في الثانية (bps)
  ④ التزامن: اتفاق المرسِل والمستقبِل على توقيت كل بت
  ⑤ التحكم بالاتصال: بدء ونهاية الاتصال الفيزيائي
🔗 Layer 2 — Data Link Layer طبقة ربط البيانات
Core Definition
The data link layer is responsible for transmitting frames from one node to the next (node-to-node delivery).

Responsibilities:

  • Framing — divides raw bits into manageable, structured data units called frames
  • Physical (MAC) Addressing — adds sender & receiver MAC address to each frame header
  • Error Control — detects errors; requests retransmission if error occurs (via trailer)
  • Flow Control — manages speed mismatch between fast sender and slow receiver
  • Access Control — determines which device can use the shared medium at a given time

Frame Structure:

Header
MAC addresses
+ control
Data
Network layer packet
Trailer
Error detection bits
Important!
The trailer (unlike other layers) contains error-detection bits, not just a header.
Arabic — طبقة ربط البيانات
مسؤولة عن نقل الـ Frames من node لـ node (القفزة بالقفزة — hop-by-hop).
Framing: تُقسّم البتات إلى وحدات منظمة تُسمى Frames لسهولة المعالجة والتحكم.
العنونة الفيزيائية (MAC): كل frame يحمل عنوان MAC المرسل والمستقبل — عنوان محلي فقط.
التحكم بالأخطاء: تكتشف الأخطاء وتطلب إعادة الإرسال (الـ Trailer يحمل bits للكشف عن الأخطاء).
التحكم بالتدفق: تمنع المرسل السريع من إغراق المستقبل البطيء.
التحكم بالوصول: تحدد من يحق له استخدام الوسط المشترك (مهم في الشبكات المحلية LAN).
🌍 Layer 3 — Network Layer طبقة الشبكة
Core Definition
The network layer is responsible for the delivery of packets from the original source to the final destination (source-to-destination), possibly crossing multiple networks.

Responsibilities:

  • Source-to-Destination Delivery — routes data across multiple networks and routers
  • Logical (IP) Addressing — assigns universal addresses that cross LAN boundaries (unlike MAC which is local only)
  • Routing — finds the best path from source to destination
  • Packet Switching — breaks data into packets and routes each independently

Why we need the Network Layer:

Key Problem
If two devices are on different networks (different LANs), MAC addresses alone are not enough — they only work locally. We need IP addresses which are universal and can cross network boundaries.
Data unit: Packet (also called Datagram)
Device: Router
Address: IP Address (32-bit in IPv4)
Arabic — طبقة الشبكة
مسؤولة عن توصيل الـ Packets من المصدر الأصلي للوجهة النهائية — حتى لو كانا على شبكات مختلفة.
لماذا نحتاجها؟ عناوين MAC محلية فقط (تعمل داخل الـ LAN). عندما نريد التواصل مع شبكة أخرى نحتاج عناوين IP التي تعبر حدود الشبكات.
التوجيه (Routing): الـ Routers تستخدم هذه الطبقة لتحديد أفضل مسار للبيانات.
Packet Switching: البيانات تُقسَّم إلى Packets، كل Packet يُوجَّه بشكل مستقل، ثم تُعاد التجميع عند الوجهة.
📦 Layer 4 — Transport Layer طبقة النقل
Core Definition
The transport layer is responsible for delivery of a message from one process to another (process-to-process delivery).

Responsibilities:

  • Port Addressing — identifies which process (application) on the machine is communicating
  • Segmentation and Reassembly — breaks large messages into segments; reassembles at destination
  • Connection Control:
    • Connectionless — UDP: fast, no guarantee (streaming, games)
    • Connection-oriented — TCP: reliable, ordered, confirmed delivery
  • End-to-End Flow Control — manages speed between processes
  • End-to-End Error Control — ensures complete, correct delivery

لا تكفي عناوين IP لتحديد التطبيق — فكل جهاز يُشغّل عدة برامج في نفس الوقت.

  • Provides user interfaces for network services
  • Services: E-mail, File transfer (FTP), Remote login (Telnet/SSH), WWW (HTTP), DNS

The OSI Model — 7 Layers

نموذج OSI السباعي — النموذج المرجعي
📐 OSI Overview نظرة عامة على OSI

The Open Systems Interconnection (OSI) model is a theoretical 7-layer model developed by ISO that describes how a protocol stack should be implemented. It is a reference model — not widely used in practice (TCP/IP is used instead).

Arabic
OSI = Open Systems Interconnection — طُوِّر من قبل منظمة ISO كنموذج مرجعي نظري.
الغرض: يصف كيف يجب أن تعمل الشبكات — لكنه لم يُطبَّق تجارياً بشكل كامل.
في الواقع: TCP/IP هو المعيار المستخدم.
OSI مفيد كأداة تعليمية لفهم مبادئ الطبقات.

🆕 The Two Extra OSI Layers الطبقتان الإضافيتان في OSI
Layer 5 — Session Layer

Network dialog controller

  • Establishes, maintains, and synchronizes communications between systems
  • Manages dialog control (who speaks when)
  • Handles session setup and teardown
  • Supports synchronization checkpoints (resume from error)
Layer 6 — Presentation Layer

Handles syntax and semantics of information exchange

  • Data translation — converts between different formats (e.g., EBCDIC ↔ ASCII)
  • Encryption / Decryption — security
  • Compression — reduces data size for transmission
Why TCP/IP Doesn't Have Them
In the TCP/IP model, the duties of Session and Presentation layers are handled by the Application layer itself (or by individual application protocols). This is why OSI has 7 layers but TCP/IP only has 5.
Arabic — الطبقتان الإضافيتان
Session Layer (طبقة الجلسة):
تتحكم في الحوار بين الأجهزة — تُنشئ الجلسات وتُحافظ عليها وتنهيها.
مثال: جلسة فيديو كول — تُنشئ الاتصال وتُزامنه وتُنهيه بشكل منظم.

Presentation Layer (طبقة العرض):
تُعالج صياغة البيانات وتنسيقها:
• الترجمة: تحويل أنظمة ترميز مختلفة (مثل ASCII ↔ Unicode)
• التشفير/فك التشفير: حماية البيانات أثناء النقل
• الضغط: تقليل حجم البيانات لتسريع النقل

في TCP/IP: هذه المهام تؤديها طبقة التطبيق مباشرة — لذلك اختفت كطبقات مستقلة.

Addresses in TCP/IP — The Critical Rules

العناوين في TCP/IP — القواعد الأساسية
🌳 Types of Addresses — Overview Tree شجرة أنواع العناوين

In TCP/IP, there are four distinct types of addresses, each operating at a different layer and serving a different purpose. The diagram below shows how they branch from the single concept of "Addresses".

① Physical Addresses (MAC)
  • Used by the Data Link Layer (Layer 2) + Physical Layer (Layer 1)
  • Also called MAC address (Media Access Control)
  • Size: 48 bits (6 bytes) — written as 6 hex pairs
  • Example: 07:01:02:01:2C:4B
  • Scope: Local only — has no meaning outside its LAN
  • Changes at every router hop ✅
عنوان MAC: يُعرّف الجهاز على الشبكة المحلية. يتغير في كل router.
② Logical Addresses (IP)
  • Used by the Network Layer (Layer 3)
  • Also called IP address (Internet Protocol)
  • Size: 32 bits in IPv4 — written as 4 decimal octets
  • Example: 192.168.1.1
  • Scope: Global — works across all networks
  • Stays the same end-to-end ❌ (no change)
عنوان IP: يُعرّف الجهاز على الإنترنت. يبقى ثابتاً طول الرحلة.
③ Port Addresses
  • Used by the Transport Layer (Layer 4)
  • Identifies a specific process (application) on a machine
  • Size: 16 bits — range 0 to 65535
  • Examples: 80 (HTTP), 443 (HTTPS), 25 (SMTP)
  • Scope: Per machine — differentiates apps on same IP
  • Stays the same end-to-end ❌ (no change)
رقم المنفذ: يُحدد التطبيق (البرنامج) على الجهاز. يبقى ثابتاً طول الرحلة.
④ Specific Addresses
  • Used by the Application Layer (Layer 5)
  • Human-readable names and identifiers
  • Types: URLs, email addresses, domain names
  • Example: www.google.com or user@mail.com
  • Translated to IP via DNS (Domain Name System)
  • Scope: Application-specific — meaningful to users
العناوين المحددة: أسماء مقروءة للإنسان مثل URLs وعناوين البريد. تُترجَم لـ IP عبر DNS.

شرح بالعربي — أنواع العناوين الأربعة
أنواع العناوين في TCP/IP أربعة، كل نوع يعمل في طبقة مختلفة:

① العناوين الفيزيائية (MAC): تعمل في طبقة Data Link. حجمها 48 بت. محلية فقط. تتغير في كل router.
② العناوين المنطقية (IP): تعمل في طبقة Network. حجمها 32 بت. عالمية. لا تتغير طول الرحلة.
③ عناوين المنافذ (Port): تعمل في طبقة Transport. حجمها 16 بت. تُحدد التطبيق. لا تتغير طول الرحلة.
④ العناوين المحددة (Specific): تعمل في طبقة Application. أسماء مقروءة (URL, email). تُترجَم لـ IP عبر DNS.

القاعدة للامتحان: MAC فقط هو الذي يتغير في كل hop — الباقي ثابت من المصدر للوجهة.

🔑 The Most Important Rule
Physical (MAC) addresses change at every hop. Logical (IP) addresses and port numbers remain the same from source to final destination.

✅ MAC Address Details:

  • 48-bit (6 bytes) physical address
  • Written as 12 hexadecimal digits
  • Each byte separated by colon: 07:01:02:01:2C:4B
  • Unique to each network interface card (NIC)
  • Local only — has no meaning outside its LAN

✅ Port Number Details:

  • 16-bit address — represented as one decimal number
  • Range: 0 – 65535
  • Example: Port 753 (as shown in textbook)
  • Identifies the process (application) not the machine
  • Same port used throughout entire communication
Arabic — العناوين
عنوان MAC (48 بت): عنوان فيزيائي محلي، يتغير من hop لـ hop.
مثال: عندما تُرسِل بيانات من حاسوبك للإنترنت، يتغير عنوان MAC في كل router تمر به.

عنوان IP (32 بت): عنوان منطقي عالمي، يبقى ثابتاً من المصدر للوجهة.
مثال: IP الخادم الذي تتصل به يبقى نفسه طوال الرحلة.

رقم الـ Port (16 بت): يُحدد البرنامج (العملية) على الجهاز.
مثال: المتصفح يستخدم Port 80/443، بينما البريد يستخدم Port 25.
رقم الـ Port الموضح في المادة: 753 — يُمثَّل كرقم عشري واحد.

عند الإرسال (Encapsulation):
البيانات تنزل من طبقة لطبقة — كل طبقة تُضيف Header خاص بها (مثل تغليف هدية في صناديق متداخلة).

Worked Examples from the Textbook

أمثلة محلولة من الكتاب
📘 Example 1 — Data Link Layer (Physical Addressing) مثال 1 — العنونة الفيزيائية

A node with physical address 10 sends a frame to a node with physical address 87. The two nodes are connected by a link.

Solution / Analysis
1
Both nodes are on the same link (same LAN) — so we only need Data Link layer (physical/MAC addresses)
2
The frame header contains: Source MAC: 10 Destination MAC: 87
3
The trailer contains extra bits for error detection
4
Since they're on the same physical link, no Network layer (IP) needed
Arabic
Node ذو عنوان 10 يُرسِل frame لـ Node ذو عنوان 87 على نفس الرابط.
الحل: لأنهما على نفس LAN، نحتاج فقط عناوين MAC.
Frame Header: عنوان MAC المصدر (10) + عنوان MAC الوجهة (87) + معلومات تحكم أخرى.
Frame Trailer: بتات اضافية للكشف عن الأخطاء (error detection bits).
لا حاجة لعناوين IP هنا لأنهما على نفس الشبكة المحلية.
📘 Example 2 — Network Layer (Two Different LANs) مثال 2 — شبكتان مختلفتان

We want to send data from a node with network address A and physical address 10 on one LAN, to a node with network address P and physical address 95 on another LAN.

Solution / Analysis
1
Two devices on different networks — physical (MAC) addresses alone won't work (they're local only)
2
We need logical (IP/network) addresses — they are universal and cross LAN boundaries
3
Network layer adds IP header: Source IP: A Dest IP: P
4
Data Link layer adds MAC header for each hop: changes at every router, but IP stays the same
Arabic
جهازان على شبكتين مختلفتين — لا يمكن استخدام عناوين MAC وحدها.
السبب: عناوين MAC محلية فقط، لا تعمل خارج الـ LAN الواحدة.
الحل: نستخدم عناوين IP (منطقية) التي لها صلاحية عالمية.
طبقة الشبكة تُضيف: IP المصدر (A) + IP الوجهة (P) → تبقى ثابتة في كل الرحلة.
طبقة ربط البيانات تُضيف MAC جديد في كل hop (router) للشبكة التالية.
📘 Example 3 — Transport Layer (Process-to-Process) مثال 3 — طبقة النقل

Data from upper layers has port addresses j (sending process) and k (receiving process). The data size is larger than the network layer can handle.

Solution / Analysis
1
Transport layer identifies processes: Source Port: j Dest Port: k
2
Data is too large → transport layer splits into two segments (segmentation)
3
Each segment retains both port addresses (j and k)
4
Network layer adds IP addresses (A and P) to each packet separately
5
At destination: transport layer reassembles segments in correct order
Arabic
البيانات أكبر من حجم الـ Packet — طبقة النقل تُقسّمها لـ Segments.
كل Segment يحتفظ بأرقام الـ Ports (j للإرسال، k للاستقبال).
ثم طبقة الشبكة تُضيف عناوين IP (A وP) لكل Packet على حدة.
عند الوصول: طبقة النقل تُعيد تجميع الـ Segments بالترتيب الصحيح.
📘 Example 4 — Complete Communication (Two Computers via Internet) مثال 4 — تواصل كامل عبر الإنترنت

Sending computer runs 3 processes (ports a, b, c). Receiving computer runs 2 processes (ports j, k). Process a needs to communicate with process j.

Key Observations
  • Physical (MAC) addresses change from hop to hop
  • Logical (IP) addresses remain the same from source to destination
  • Port addresses remain the same from source to destination
  • Port a identifies the sending process; port j identifies the receiving process
Arabic
جهاز الإرسال يُشغّل 3 عمليات (a, b, c) | جهاز الاستقبال يُشغّل 2 (j, k).
العملية a تتواصل مع العملية j → Port a (مصدر) إلى Port j (وجهة).
الملاحظة الأهم:
• عنوان MAC: يتغير في كل router على الطريق (hop-by-hop)
• عنوان IP: يبقى ثابتاً طول الرحلة (end-to-end)
• رقم الـ Port: يبقى ثابتاً طول الرحلة (end-to-end)

Exam Questions & Complete Solutions

أسئلة الامتحان مع الحلول الكاملة
Question 1
What is the responsibility of each TCP/IP layer? Give the data unit name for each.
✅ Answer
Physical (1): Transmit individual bits node-to-node over physical medium.
Data Link (2): Transmit frames node-to-node; framing, MAC addressing, error + flow control.
Network (3): Deliver packets source-to-destination across multiple networks; IP addressing + routing.
Transport (4): Deliver segments/messages process-to-process; port addressing, segmentation, TCP/UDP.
Application (5): Provide services (messages) to the user; HTTP, FTP, DNS, email.
Question 2
What address does each layer use? What is the size of each address type?
✅ Answer
Data Link Layer: MAC Address — 48 bits (6 bytes), written as 12 hex digits separated by colons (e.g., 07:01:02:01:2C:4B)
Network Layer: IP Address — 32 bits (IPv4)
Transport Layer: Port Number — 16 bits, represented as a single decimal number (e.g., 753)
Question 3
What are the two extra layers in OSI compared to TCP/IP? What does each do?
✅ Answer
Session Layer (Layer 5 in OSI): Network dialog controller — establishes, maintains, and synchronizes communication sessions between systems.
Presentation Layer (Layer 6 in OSI): Handles data translation (format conversion), encryption/decryption, and compression. Both are absorbed into the Application layer in TCP/IP.
Question 4
Does the MAC address change at each hop? What about IP and Port addresses?
✅ Answer
MAC address: YES — changes at every hop (router). It is local to each link.
IP address: NO — remains the same from source to destination (end-to-end).
Port number: NO — remains the same from source to destination (end-to-end).
Question 5
What is peer-to-peer communication in networking? Give an example.
✅ Answer
Peer-to-peer communication means corresponding layers on two different machines communicate logically with each other. Example: The network layer on Computer A communicates logically with the network layer on Computer B using the IP protocol — even though physically the data passes through all five layers at both ends and all intermediate nodes.
Question 6
Why is the Network layer needed when we already have the Data Link layer?
✅ Answer
The Data Link layer uses MAC addresses which are local — they only work within a single LAN (no meaning outside). If two devices are on different networks, we cannot use MAC addresses to route between them. The Network layer provides logical (IP) addresses that are universal and can cross LAN boundaries, enabling source-to-destination delivery across multiple networks.
Question 7
What is encapsulation? Describe the process as data travels from Application to Physical layer.
✅ Answer
Encapsulation is the process where each layer adds its own header (control information) to the data received from the layer above.
Process:
1. Application creates a message (DATA)
2. Transport adds TCP/UDP header → Segment [TCP | DATA]
3. Network adds IP header → Packet [IP | TCP | DATA]
4. Data Link adds MAC header + trailer → Frame [MAC-H | IP | TCP | DATA | Trailer]
5. Physical converts to bits and transmits
Question 8
What is the difference between TCP and UDP at the Transport layer?
✅ Answer
TCP (Transmission Control Protocol): Connection-oriented, reliable delivery, flow control, error control, ordered segments. Used for: HTTP, FTP, email — where accuracy matters.
UDP (User Datagram Protocol): Connectionless, no delivery guarantee, minimal overhead, fast. Used for: streaming video, VoIP, DNS, gaming — where speed matters more than reliability.
Question 9
What is the difference between the Internet Model (TCP/IP) and the OSI model? Which is actually used?
✅ Answer
TCP/IP: 5 layers, practical model, developed from ARPANET, widely used in the real Internet.
OSI: 7 layers, theoretical reference model, developed by ISO, never fully implemented commercially.
The TCP/IP model is actually used in practice. OSI is used as a teaching/reference framework only.
Question 10 — Scenario
You send an email from your PC in Jordan to a server in the USA. At which layer does each of the following happen? (a) Your email text is created. (b) The email is split into segments. (c) Each segment gets an IP address. (d) Each packet gets a MAC address for the next hop. (e) Bits are sent over the cable.
✅ Answer
(a) Application Layer — message created by email protocol (SMTP)
(b) Transport Layer — segmentation and port addressing
(c) Network Layer — IP header added with your IP and server's IP
(d) Data Link Layer — MAC header added for the local next-hop router
(e) Physical Layer — electrical/optical signals transmitted

📚 Chapter 2 — Complete Review Sheet ورقة المراجعة النهائية الشاملة

TCP/IP — 5 Layers (Bottom → Top)
  1. Physical → Bits → Node-to-Node → No address
  2. Data Link → Frames → Node-to-Node → MAC (48-bit)
  3. Network → Packets → Host-to-Host → IP (32-bit)
  4. Transport → Segments → Process-to-Process → Port (16-bit)
  5. Application → Messages → User-to-User → URL/App
OSI Extra Layers
  • Session (5): Dialog control & sync
  • Presentation (6): Translation, encryption, compression
Critical Rules
  • MAC changes every hop ✓
  • IP & Port stay same end-to-end ✓
  • Encapsulation = add headers going down ✓
  • Decapsulation = remove headers going up ✓

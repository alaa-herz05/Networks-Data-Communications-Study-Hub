IPv4 — Introduction & Basics

المقدمة وأساسيات عنوان IPv4
What is an IPv4 Address? ما هو عنوان IPv4؟
  • An IPv4 address is a 32-bit identifier used in the IP layer of TCP/IP.
  • It uniquely and universally identifies each device's connection to the Internet.
  • Total address space: 232 = 4,294,967,296 (~4.3 billion addresses).
  • Written in dotted-decimal notation: four decimal bytes separated by dots.
  • Range: 0.0.0.0 (minimum) to 255.255.255.255 (maximum).
🔑 Key Rule — القاعدة الأساسية
IPv4 = 32 bits = 4 bytes = 4 octets. Each byte is 8 bits → value from 0 to 255.
Arabic
عنوان IPv4 هو 32 بت مقسَّم إلى 4 أجزاء كل جزء 8 بت (بايت). نكتبه كأربعة أرقام عشرية تفصلها نقاط مثل 192.168.1.1. كل رقم يتراوح بين 0 و255. المساحة الكلية للعناوين = 232 = 4,294,967,296 عنوان.

Notation Systems — Binary, Decimal, Hexadecimal

أنظمة التمثيل: ثنائي، عشري، سادس عشري
Three Representations ثلاث طرق لكتابة العنوان
Same address — three formats
Binary (Base 2): 10000001 00001011 00001011 11101111 Dotted-Decimal: 129.11.11.239 Hexadecimal (Base16): 0x810B0BEF

Binary → Decimal conversion: Each group of 8 bits → one decimal number (0–255).

Binary → Hex conversion: Each group of 4 bits → one hex digit (0–F).

Dotted-Decimal: Most common format. e.g. 129.11.11.239

Hex: Prefixed with 0x or subscript 16. Used in some protocols.

Arabic
ثلاث طرق لكتابة نفس العنوان:
• ثنائي (Binary): مجموعات من 0 و1، كل مجموعة 8 أرقام.
• عشري منقط (Dotted-Decimal): الأكثر استخداماً — أرقام من 0-255 مفصولة بنقاط.
• سادس عشري (Hex): كل 4 بتات = رقم hex، يُستخدم أحياناً في البروتوكولات.
Bitwise Operations — AND, OR, NOT العمليات البتية

These operations are essential for finding first and last addresses.

Operation Symbol Rule Example بالعربي
NOT ~ Flips each bit: 0→1, 1→0 ~11000000 = 00111111 عكس كل بت
AND & 1 AND 1=1, anything else=0 10100101 AND 11110000 = 10100000 1 فقط إذا كلاهما 1
OR | 0 OR 0=0, anything else=1 10100000 OR 00001111 = 10101111 0 فقط إذا كلاهما 0
🔑 Usage in IPv4
First address = address AND mask   |   Last address = address OR (NOT mask)
Arabic
العمليات البتية الثلاث ضرورية لحل مسائل IPv4:
• NOT: يعكس كل بت (0 يصبح 1، و1 يصبح 0).
• AND: الناتج 1 فقط إذا كلا البتين 1 — نستخدمه لإيجاد أول عنوان.
• OR: الناتج 0 فقط إذا كلا البتين 0 — نستخدمه مع NOT للماسك لإيجاد آخر عنوان.

Classful Addressing — الفئات A, B, C, D, E

العنونة ذات الفئات — كيف تُقسَّم مساحة العناوين
The Five Classes الفئات الخمس

IPv4 address space is divided into 5 classes based on the first byte value (or the leading bits).

Class Leading Bits 1st Byte Range Network Bits (n) Host Bits Block Size (Addresses) Purpose بالعربي
A 0 0 – 127 8 24 224 = 16,777,216 Very large orgs مؤسسات ضخمة جداً
B 10 128 – 191 16 16 216 = 65,536 Medium-large orgs مؤسسات متوسطة-كبيرة
C 110 192 – 223 24 8 28 = 256 Small orgs مؤسسات صغيرة
D 1110 224 – 239 — — One block Multicasting البث المتعدد
E 11110 240 – 255 — — One block Reserved (future) محجوز للمستقبل
🔑 Quick Identification Rule — قاعدة تحديد الفئة
Look at the first byte:   0–127 = A  |  128–191 = B  |  192–223 = C  |  224–239 = D  |  240–255 = E
Arabic
الفئات الخمس تُحدَّد من خلال البايت الأول:
• A (0-127): شبكة ضخمة جداً — 16 مليون عنوان في الكتلة الواحدة. بتات الشبكة = 8.
• B (128-191): متوسط — 65,536 عنوان. بتات الشبكة = 16.
• C (192-223): صغيرة — 256 عنوان فقط. بتات الشبكة = 24.
• D (224-239): Multicast — بث متعدد، كتلة واحدة.
• E (240-255): محجوز للمستقبل، لا يُستخدم حالياً.

Classful First/Last Address Rules
Class A (n=8): First = X.0.0.0 Last = X.255.255.255 Class B (n=16): First = X.Y.0.0 Last = X.Y.255.255 Class C (n=24): First = X.Y.Z.0 Last = X.Y.Z.255 Number of addresses = 2^(32 - n) First address: keep leftmost n bits, set hostid all to 0s Last address: keep leftmost n bits, set hostid all to 1s

Classless Addressing — CIDR (Slash Notation)

العنونة بدون فئات — CIDR والـ Slash Notation
Why Classless? لماذا ابتُكر CIDR؟
  • Classful addressing wastes millions of addresses: a small org gets Class C (256) but may need 300 → gets Class B (65,536) → 65,236 wasted.
  • CIDR = Classless Inter-Domain Routing — uses variable-length blocks defined by a prefix length (1–32).
  • A block is defined by: one address in the block + the prefix length.
  • Prefix length can be 1 to 32 — completely flexible allocation.
Arabic
العنونة الفئوية كانت تُهدر ملايين العناوين (كتلة Class A = 16 مليون، والمؤسسة قد تحتاج 1000 فقط!). CIDR جاء ليحل هذه المشكلة بتخصيص كتل متغيرة الحجم بدقة حسب الحاجة. نكتب الكتلة كـ عنوان/prefix مثل: 192.168.1.0/25 أي أول 25 بت هي الشبكة.

IPv4 Figure 5.27
Exam focus: Classless addressing allocates variable-size blocks instead of fixed class A/B/C blocks. العنونة بدون فئات تخصص كتل بأحجام مختلفة بدل الالتزام بأحجام A/B/C الثابتة.

Arabic
في CIDR، العنوان يُكتب بصيغة: عنوان IP / طول البادئة مثل 192.168.1.64/26.
• Prefix (البادئة): بتات الشبكة — يُحدِّد الشبكة.
• Suffix (اللاحقة): 32 - prefix — يُحدِّد المضيف.
• عدد العناوين = 2^(32-n).
• القناع (Mask) = n من الـ 1s ثم (32-n) من الـ 0s.
• أول عنوان = AND مع القناع.
• آخر عنوان = OR مع عكس (NOT) القناع.

Step-by-Step: How to Solve Any IPv4 Problem

خطوات الحل — كيف تحل أي مسألة IPv4
Method A — Classful (No slash given) الطريقة أ — العنونة الفئوية
🗺️ Steps for Classful Problems
1
Look at the first byte of the address to identify the class.
0-127→A, 128-191→B, 192-223→C, 224-239→D, 240-255→E
2
Determine n (network bits): A→8, B→16, C→24.
3
Calculate number of addresses = 2(32−n)
4
Find first address: keep left n bits, set remaining bits to all 0s.
5
Find last address: keep left n bits, set remaining bits to all 1s (255 for each full byte).
Arabic
خطوات حل المسائل الفئوية (Classful):
① انظر للبايت الأول → حدِّد الفئة (A/B/C/D/E).
② حدِّد n (بتات الشبكة): A=8، B=16، C=24.
③ عدد العناوين = 2^(32-n).
④ أول عنوان: احتفظ بـ n بت من اليسار، اجعل الباقي = أصفار.
⑤ آخر عنوان: احتفظ بـ n بت من اليسار، اجعل الباقي = واحدات (255 لكل بايت كامل).
Method B — Classless CIDR (Slash given) الطريقة ب — العنونة بدون فئات
🗺️ Steps for CIDR Problems
1
Extract the prefix length n from the slash notation (e.g., /27 → n=27).
2
Calculate number of addresses = 2(32−n)
3
Build the subnet mask: n ones followed by (32−n) zeros, written in dotted-decimal.
4
Find first address = given address AND subnet mask (byte by byte).
5
Find last address: Invert the mask (NOT), then OR with first address.
💡 Shortcut for boundary bytes:
If a byte has all 1s in the mask → keep the address byte as-is (for first address).
If a byte has all 0s in the mask → set it to 0 (first) or 255 (last).
If a byte is mixed → do the actual AND/OR bit by bit.
Arabic
خطوات حل المسائل CIDR:
① استخرج n من الـ Slash (مثل /27 → n=27).
② عدد العناوين = 2^(32-27) = 2^5 = 32.
③ ابنِ القناع: 27 رقم واحد + 5 أصفار → 11111111.11111111.11111111.11100000 → 255.255.255.224.
④ أول عنوان = العنوان AND القناع (بايت بايت).
⑤ آخر عنوان = عكس القناع (NOT) ثم OR مع أول عنوان.

الاختصار السهل: البايت اللي القناع فيه 255 → يبقى كما هو. البايت اللي القناع فيه 0 → يصبح 0 للأول و255 للأخير. البايت المختلط → احسبه بشكل ثنائي.

البايت المختلط في /20 هو البايت الثالث: 11110000 بالثنائي.

Fully Solved Examples — الأمثلة المحلولة بالتفصيل

جميع الأمثلة محلولة خطوة بخطوة
Example 1 — Classful, Class A | Given: 73.22.17.25
1
First byte = 73 → range 0–127 → Class A → n = 8
2
Number of addresses = 2(32−8) = 224 = 16,777,216
3
First address: keep 1st byte (73), set rest to 0 → 73.0.0.0
4
Last address: keep 1st byte (73), set rest to 255 → 73.255.255.255
Arabic Explanation
البايت الأول 73 يقع بين 0-127 ← Class A. n=8 (8 بتات للشبكة). عدد العناوين = 2^24 ≈ 16 مليون. أول عنوان: البايت الأول ثابت والباقي أصفار = 73.0.0.0. آخر عنوان: البايت الأول ثابت والباقي 255 = 73.255.255.255.
Example 2 — Classful, Class B | Given: 180.8.17.9
1
First byte = 180 → range 128–191 → Class B → n = 16
2
Number of addresses = 2(32−16) = 216 = 65,536
3
First address: keep first 2 bytes (180.8), set bytes 3&4 to 0 → 180.8.0.0
4
Last address: keep first 2 bytes (180.8), set bytes 3&4 to 255 → 180.8.255.255
Arabic Explanation
180 بين 128-191 ← Class B. n=16. عدد العناوين = 2^16 = 65,536. أول عنوان: البايتان الأولان ثابتان والباقي أصفار = 180.8.0.0. آخر عنوان: البايتان الأولان ثابتان والباقي 255 = 180.8.255.255.
Example 3 — Classful, Class C | Given: 200.11.8.45
1
First byte = 200 → range 192–223 → Class C → n = 24
2
Number of addresses = 2(32−24) = 28 = 256
3
First address: keep first 3 bytes (200.11.8), set byte 4 to 0 → 200.11.8.0
4
Last address: keep first 3 bytes (200.11.8), set byte 4 to 255 → 200.11.8.255
Arabic Explanation
200 بين 192-223 ← Class C. n=24. عدد العناوين = 2^8 = 256. أول عنوان: ثلاث بايتات الأولى ثابتة والرابع = 0 → 200.11.8.0. آخر عنوان: الرابع = 255 → 200.11.8.255.
Example 4 — CIDR /27 | Given: 167.199.170.82/27
1
Prefix = n = 27 → Host bits = 32 − 27 = 5
2
Number of addresses = 25 = 32
3
Build mask: 27 ones + 5 zeros
11111111.11111111.11111111.11100000
Mask = 255.255.255.224
4
First address = address AND mask:
167.199.170.82 → 10100111.11000111.10101010.01010010 AND 255.255.255.224 → 11111111.11111111.11111111.11100000 ───────────────────────────────────────────────────────── → 10100111.11000111.10101010.01000000 Result = 167.199.170.64
First address = 167.199.170.64/27
5
Last address = first address OR (NOT mask):
NOT mask: 00000000.00000000.00000000.00011111 = 0.0.0.31
167.199.170.64 → 10100111.11000111.10101010.01000000 OR 0.0.0.31 → 00000000.00000000.00000000.00011111 ───────────────────────────────────────────────────────── → 10100111.11000111.10101010.01011111 Result = 167.199.170.95
Last address = 167.199.170.95/27
Arabic Explanation — شرح /27 بالتفصيل
n=27 → host bits = 5 → عدد العناوين = 2^5 = 32.
القناع: 27 واحد + 5 أصفار = 255.255.255.224 (البايت الأخير = 11100000 = 224).
أول عنوان: AND بين 82 (01010010) والـ 224 (11100000) = 64 (01000000) → 167.199.170.64.
عكس القناع: NOT 224 = NOT 11100000 = 00011111 = 31.
آخر عنوان: OR بين 64 (01000000) و31 (00011111) = 95 (01011111) → 167.199.170.95.
إذاً الكتلة من 167.199.170.64 إلى 167.199.170.95 (32 عنوان).
Example 5 — CIDR /24 | Given: 17.63.110.114/24
1
n = 24 → Host bits = 8
2
Number of addresses = 28 = 256
3
Mask = 255.255.255.0 (24 ones, 8 zeros)
4
First address: bytes 1-3 unchanged (mask all 1s), byte 4 → 0: 17.63.110.0/24
5
Last address: bytes 1-3 unchanged, byte 4 → 255: 17.63.110.255/24
Arabic
/24 مثل Class C: البايتات 1-3 ثابتة، البايت 4 فقط يتغير. أول عنوان: آخر بايت = 0. آخر عنوان: آخر بايت = 255. سهل!
Example 6 — CIDR /20 (Mixed byte!) | Given: 110.23.120.14/20
1
n = 20 → Host bits = 32 − 20 = 12
2
Number of addresses = 212 = 4,096
3
Mask: 20 ones + 12 zeros
11111111.11111111.11110000.00000000 = 255.255.240.0
Byte 3 is mixed: 11110000 = 240
4
First address: Byte 3 = 120 AND 240:
120 = 01111000 240 = 11110000 AND = 01110000 = 112
First address = 110.23.112.0/20
5
Last address: NOT mask byte 3 = NOT 240 = 15 (00001111). OR with 112: 112 OR 15 = 127
Last address = 110.23.127.255/20
Arabic Explanation — /20 البايت المختلط
n=20، أي 20 بت للشبكة. البايتان الأولان (bytes 1,2) كلهم في الشبكة ← يبقيان كما هما (110 و23). البايت 3 مختلط: 4 بتات للشبكة + 4 للمضيف.
القناع للبايت 3 = 11110000 = 240. 120 AND 240 = 01111000 AND 11110000 = 01110000 = 112.
عكس القناع للبايت 3 = 00001111 = 15. 112 OR 15 = 01111000 OR 00001111 = 01111111 = 127.
البايت 4 = 0 للأول و255 للأخير.
النتيجة: 110.23.112.0 إلى 110.23.127.255 (4096 عنوان).

Q4 — Classful vs Classless Difference
Explain the difference between classful and classless addressing.

Answer:
Classful: Fixed block sizes (Class A=16M, B=65K, C=256). Simple but wastes millions of addresses.
Classless (CIDR): Variable-length blocks using /n prefix (1–32). Allocates exactly what's needed → far less waste. Identified by slash notation (e.g., /27).
Arabic
Classful: كتل ثابتة الحجم (A=16M، B=65K، C=256) — بسيطة لكن تهدر ملايين العناوين.
Classless CIDR: كتل متغيرة الحجم بـ /n — تُخصَّص الكتلة بدقة حسب الحاجة لتقليل الهدر.
Q5 — Network mask and block info, CIDR /16

• Number of addresses = 2(32-16) = 216 = 65,536
• First: Keep bytes 1-2 (130.11), set bytes 3-4 to 0 → 130.11.0.0/16
• Last: Keep bytes 1-2, set bytes 3-4 to 255 → 130.11.255.255/16

Q6 — Full CIDR solution /27

Answer:
• n = 27, host bits = 5
• Number of addresses = 25 = 32
• Mask = 255.255.255.224 (last byte: 11100000 = 224)

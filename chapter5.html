IPv4 — Introduction & Basics

المقدمة وأساسيات عنوان IPv4
What is an IPv4 Address? ما هو عنوان IPv4؟
  • An IPv4 address is a 32-bit identifier used in the IP layer of TCP/IP.
  • It uniquely and universally identifies each device's connection to the Internet.
  • Total address space: 232 = 4,294,967,296 (~4.3 billion addresses).
  • Written in dotted-decimal notation: four decimal bytes separated by dots.
  • Range: 0.0.0.0 (minimum) to 255.255.255.255 (maximum).
🔑 Key Rule — القاعدة الأساسية
IPv4 = 32 bits = 4 bytes = 4 octets. Each byte is 8 bits → value from 0 to 255.
Arabic
عنوان IPv4 هو 32 بت مقسَّم إلى 4 أجزاء كل جزء 8 بت (بايت). نكتبه كأربعة أرقام عشرية تفصلها نقاط مثل 192.168.1.1. كل رقم يتراوح بين 0 و255. المساحة الكلية للعناوين = 232 = 4,294,967,296 عنوان.

Notation Systems — Binary, Decimal, Hexadecimal

أنظمة التمثيل: ثنائي، عشري، سادس عشري
Three Representations ثلاث طرق لكتابة العنوان
Same address — three formats
Binary (Base 2): 10000001 00001011 00001011 11101111 Dotted-Decimal: 129.11.11.239 Hexadecimal (Base16): 0x810B0BEF

Binary → Decimal conversion: Each group of 8 bits → one decimal number (0–255).

Binary → Hex conversion: Each group of 4 bits → one hex digit (0–F).

Dotted-Decimal: Most common format. e.g. 129.11.11.239

Hex: Prefixed with 0x or subscript 16. Used in some protocols.

Arabic
ثلاث طرق لكتابة نفس العنوان:
• ثنائي (Binary): مجموعات من 0 و1، كل مجموعة 8 أرقام.
• عشري منقط (Dotted-Decimal): الأكثر استخداماً — أرقام من 0-255 مفصولة بنقاط.
• سادس عشري (Hex): كل 4 بتات = رقم hex، يُستخدم أحياناً في البروتوكولات.

Classful Addressing — الفئات A, B, C, D, E

العنونة ذات الفئات — كيف تُقسَّم مساحة العناوين
The Five Classes الفئات الخمس

IPv4 address space is divided into 5 classes based on the first byte value (or the leading bits).

Class Leading Bits 1st Byte Range Network Bits (n) Host Bits Block Size (Addresses) Purpose بالعربي
A 0 0 – 127 8 24 224 = 16,777,216 Very large orgs مؤسسات ضخمة جداً
B 10 128 – 191 16 16 216 = 65,536 Medium-large orgs مؤسسات متوسطة-كبيرة
C 110 192 – 223 24 8 28 = 256 Small orgs مؤسسات صغيرة
D 1110 224 – 239 — — One block Multicasting البث المتعدد
E 11110 240 – 255 — — One block Reserved (future) محجوز للمستقبل
🔑 Quick Identification Rule — قاعدة تحديد الفئة
Look at the first byte:   0–127 = A  |  128–191 = B  |  192–223 = C  |  224–239 = D  |  240–255 = E
Arabic
الفئات الخمس تُحدَّد من خلال البايت الأول:
• A (0-127): شبكة ضخمة جداً — 16 مليون عنوان في الكتلة الواحدة. بتات الشبكة = 8.
• B (128-191): متوسط — 65,536 عنوان. بتات الشبكة = 16.
• C (192-223): صغيرة — 256 عنوان فقط. بتات الشبكة = 24.
• D (224-239): Multicast — بث متعدد، كتلة واحدة.
• E (240-255): محجوز للمستقبل، لا يُستخدم حالياً.

Classless Addressing — CIDR (Slash Notation)

العنونة بدون فئات — CIDR والـ Slash Notation
Why Classless? لماذا ابتُكر CIDR؟
  • Classful addressing wastes millions of addresses: a small org gets Class C (256) but may need 300 → gets Class B (65,536) → 65,236 wasted.
  • CIDR = Classless Inter-Domain Routing — uses variable-length blocks defined by a prefix length (1–32).
  • A block is defined by: one address in the block + the prefix length.
  • Prefix length can be 1 to 32 — completely flexible allocation.
Arabic
العنونة الفئوية كانت تُهدر ملايين العناوين (كتلة Class A = 16 مليون، والمؤسسة قد تحتاج 1000 فقط!). CIDR جاء ليحل هذه المشكلة بتخصيص كتل متغيرة الحجم بدقة حسب الحاجة. نكتب الكتلة كـ عنوان/prefix مثل: 192.168.1.0/25 أي أول 25 بت هي الشبكة.

في CIDR، العنوان يُكتب بصيغة: عنوان IP / طول البادئة مثل 192.168.1.64/26.
• Prefix (البادئة): بتات الشبكة — يُحدِّد الشبكة.
• Suffix (اللاحقة): 32 - prefix — يُحدِّد المضيف.
• عدد العناوين = 2^(32-n).
• القناع (Mask) = n من الـ 1s ثم (32-n) من الـ 0s.
• أول عنوان = AND مع القناع.
• آخر عنوان = OR مع عكس (NOT) القناع.

Step-by-Step: How to Solve Any IPv4 Problem

خطوات الحل — كيف تحل أي مسألة IPv4
Method A — Classful (No slash given) الطريقة أ — العنونة الفئوية
🗺️ Steps for Classful Problems
1
Look at the first byte of the address to identify the class.
0-127→A, 128-191→B, 192-223→C, 224-239→D, 240-255→E
2
Determine n (network bits): A→8, B→16, C→24.
3
Calculate number of addresses = 2(32−n)
4
Find first address: keep left n bits, set remaining bits to all 0s.
5
Find last address: keep left n bits, set remaining bits to all 1s (255 for each full byte).
Arabic
خطوات حل المسائل الفئوية (Classful):
① انظر للبايت الأول → حدِّد الفئة (A/B/C/D/E).
② حدِّد n (بتات الشبكة): A=8، B=16، C=24.
③ عدد العناوين = 2^(32-n).
④ أول عنوان: احتفظ بـ n بت من اليسار، اجعل الباقي = أصفار.
⑤ آخر عنوان: احتفظ بـ n بت من اليسار، اجعل الباقي = واحدات (255 لكل بايت كامل).
Method B — Classless CIDR (Slash given) الطريقة ب — العنونة بدون فئات
🗺️ Steps for CIDR Problems
1
Extract the prefix length n from the slash notation (e.g., /27 → n=27).
2
Calculate number of addresses = 2(32−n)
3
Build the subnet mask: n ones followed by (32−n) zeros, written in dotted-decimal.
4
Find first address = given address AND subnet mask (byte by byte).
5
Find last address: Invert the mask (NOT), then OR with first address.
💡 Shortcut for boundary bytes:
If a byte has all 1s in the mask → keep the address byte as-is (for first address).
If a byte has all 0s in the mask → set it to 0 (first) or 255 (last).
If a byte is mixed → do the actual AND/OR bit by bit.
Arabic
خطوات حل المسائل CIDR:
① استخرج n من الـ Slash (مثل /27 → n=27).
② عدد العناوين = 2^(32-27) = 2^5 = 32.
③ ابنِ القناع: 27 رقم واحد + 5 أصفار → 11111111.11111111.11111111.11100000 → 255.255.255.224.
④ أول عنوان = العنوان AND القناع (بايت بايت).
⑤ آخر عنوان = عكس القناع (NOT) ثم OR مع أول عنوان.

الاختصار السهل: البايت اللي القناع فيه 255 → يبقى كما هو. البايت اللي القناع فيه 0 → يصبح 0 للأول و255 للأخير. البايت المختلط → احسبه بشكل ثنائي.

Three-Level Addressing — Subnetting العنونة ثلاثية المستوى — التقسيم للشبكات الفرعية

Subnetting splits a classful network block into smaller subnets by borrowing bits from the hostid. This creates a three-level hierarchy:

Address Structure — بنية العنوان
[ netid (class-fixed) | subnetid (borrowed) | hostid (remaining) ] |<--- Original netid --->| borrowed bits |<--- new hostid --->| Subnet mask for subnetting: Original class mask + borrowed bits set to 1 + remaining 0s
  • The subnet mask (or default mask + extra bits) identifies the subnetwork.
  • Each subnet behaves like a miniature independent network.
  • Routers inside the organization use the full subnet mask; outside routers see only the classful netid.
  • Formula for number of subnets: 2s where s = borrowed bits
  • Formula for hosts per subnet: 2h − 2 where h = remaining hostid bits
🔑 Subnetting Rule
Total bits = netid bits + subnet bits + host bits = 32
Subnets = 2^(subnet bits) | Hosts/subnet = 2^(host bits) − 2
Arabic — التقسيم إلى شبكات فرعية
Subnetting يعني أخذ شبكة فئوية كبيرة وتقسيمها إلى شبكات فرعية أصغر عن طريق اقتراض بتات من جزء الـ hostid. هذا يخلق 3 مستويات: (netid | subnetid | hostid). القناع الجديد يحتوي على بتات الـ netid الأصلية + البتات المقترضة (كلها 1) + باقي بتات الـ hostid (كلها 0).

مثال: شبكة Class B تأخذ شبكة واحدة 65,536 عنوان. بتقسيمها باقتراض 8 بتات: نحصل على 2⁸ = 256 شبكة فرعية، كل شبكة تحتوي على 2⁸ − 2 = 254 مضيف.

Supernetting — Classless Blocks & Address Aggregation الدمج — توحيد الشبكات (Supernetting)

Supernetting is the opposite of subnetting: multiple small classful networks are combined into one larger classless block. This is the foundation of CIDR (Classless Inter-Domain Routing).

  • Reduces the number of routing table entries (route aggregation / summarization).
  • A group of consecutive Class C networks can be announced as one CIDR prefix.
  • The block must be a power of 2 in size and the first address must be divisible by the block size.
  • The supernet mask has fewer 1-bits than the default class mask (shorter prefix).
Supernetting Rule — قاعدة الدمج
Block size = 2^k (k = number of Class C networks combined) Prefix length = 24 − k (for Class C supernetting) First address of block must be divisible by block_size
Example — Supernet 4 Class C blocks دمج 4 شبكات Class C
1Networks: 205.16.0.0, 205.16.1.0, 205.16.2.0, 205.16.3.0
2Block size = 4 → prefix reduction = 2 bits → new prefix = /22
3Supernet address: 205.16.0.0/22
4Total addresses = 4 × 256 = 1024
5Check: 205.16.0.0 → last octet of 3rd byte = 0, divisible by 4 ✓
Arabic
ندمج 4 شبكات Class C → حجم الكتلة = 4 → نخفض البادئة بـ 2 بت → بادئة جديدة = /22. عنوان الكتلة الموحَّدة = 205.16.0.0/22. عدد العناوين الكلي = 4 × 256 = 1024 عنوان. الشرط: العنوان الأول يجب أن يكون قابلاً للقسمة على حجم الكتلة (0 ÷ 4 = 0 ✓).
Address Depletion Problem مشكلة نضوب العناوين

IPv4's 32-bit address space was sufficient when designed in the 1970s, but several factors led to address exhaustion:

  • Classful waste: Organizations received entire Class A or B blocks but used only a fraction.
  • Exponential Internet growth: The number of connected devices grew far beyond predictions.
  • Inefficient allocation: Large chunks were given to universities, military, and early companies.

Solutions adopted:

  • CIDR — Classless routing reduces waste, allows variable-length blocks.
  • NAT — Many private devices share one public IP address.
  • IPv6 — Long-term solution: 128-bit addresses (340 undecillion addresses).
Arabic — نضوب عناوين IPv4

  • العناوين الخاصة: 10.x.x.x و 172.16–31.x.x و 192.168.x.x — للاستخدام الداخلي فقط، تُترجم بواسطة NAT للوصول للإنترنت.
Limited Broadcast vs Directed Broadcast البث المحدود مقابل البث الموجَّه
  • Limited broadcast (255.255.255.255): Sent to all hosts on the local network segment. Routers do not forward this packet. Used by DHCP requests.
  • Directed broadcast (e.g., 192.168.1.255): Sent to all hosts in a specific remote network. The last address of any block. Routers may forward it, but many block it for security.
  • Network address (e.g., 192.168.1.0): The first address of any block — identifies the network itself, not assigned to any host.
🔑 Key Rule
First address = Network identifier (not assignable). Last address = Directed broadcast (not assignable). Usable hosts = block_size − 2.
Arabic

البث الموجَّه: آخر عنوان في أي كتلة (مثال: 192.168.1.255 للشبكة 192.168.1.0/24) — يمكن للموجِّه تمريره للشبكة البعيدة.
عنوان الشبكة: أول عنوان في الكتلة — لا يُعطى لأي جهاز، يُعرِّف الشبكة نفسها.
Loopback Address عنوان حلقة الاسترداد
  • The entire 127.0.0.0/8 block is reserved for loopback.
  • Most commonly used: 127.0.0.1 (also called localhost).
  • A packet sent to any 127.x.x.x address never leaves the machine — it is immediately looped back to the network stack.
  • Used to test whether the TCP/IP stack is working correctly on the local machine.
  • Ping 127.0.0.1 → if it replies, TCP/IP is functional.
Arabic — حلقة الاسترداد (Loopback)
الكتلة كاملة 127.0.0.0/8 محجوزة للـ loopback. الأكثر استخداماً: 127.0.0.1 المعروف بـ "localhost". أي حزمة تُرسَل لهذا العنوان لا تغادر الجهاز أبداً — ترجع فوراً للمكدس. الاستخدام: اختبار أن بروتوكولات TCP/IP تعمل بشكل صحيح على الجهاز المحلي.

NAT — Network Address Translation

ترجمة عناوين الشبكة — NAT
What is NAT? ما هو NAT؟

NAT (Network Address Translation) is a technique that allows multiple devices with private IP addresses to share a single public IP address when accessing the Internet.

  • Defined in RFC 3022.
  • Implemented in the router/gateway at the boundary between a private network and the Internet.
  • Greatly extends the effective lifetime of IPv4 by multiplying the use of each public address.
  • Provides a basic security benefit: internal hosts are hidden from the outside world.
🔑 Core Concept
Private IP ↔ Public IP translation happens at the router. The outside world sees only the public IP; internal structure is invisible.
Arabic — ما هو NAT؟
NAT هو تقنية تسمح لعدة أجهزة تمتلك عناوين خاصة (مثل 192.168.x.x) بمشاركة عنوان عام واحد للاتصال بالإنترنت. يعمل NAT على الموجِّه/البوابة عند حدود الشبكة الخاصة. هو المسؤول الأول عن إطالة عمر IPv4 رغم نضوب عناوينه.
How NAT Works — Translation Table كيف يعمل NAT — جدول الترجمة

The NAT router maintains a translation table mapping each internal (private IP, port) pair to the (public IP, port) pair used on the Internet:

DirectionPrivate SidePublic SideAction
Outbound →192.168.1.5 : 1234203.0.113.1 : 5001Replace source IP+port
Inbound ←192.168.1.5 : 1234203.0.113.1 : 5001Replace dest IP+port

Step-by-step outbound flow:

1Internal host (192.168.1.5) sends a packet destined for 8.8.8.8:80.
2NAT router intercepts the packet. Source = 192.168.1.5:1234.
3Router replaces source with public IP+new port: 203.0.113.1:5001. Records mapping in table.
4Packet goes to the Internet with source = 203.0.113.1:5001.
5Reply arrives at 203.0.113.1:5001. Router looks up table → forwards to 192.168.1.5:1234.
Arabic — كيف يعمل NAT
الموجِّه يحتفظ بـ جدول الترجمة: كل جهاز داخلي يحصل على رقم منفذ عام مؤقت. عند الإرسال: الموجِّه يستبدل العنوان الخاص بالعنوان العام ويسجِّل الترجمة. عند الرد: الموجِّه يبحث في الجدول ويُعيد توجيه الحزمة للجهاز الداخلي الصحيح. الأجهزة الخارجية ترى فقط عنوان الموجِّه العام.

Dynamic NAT: مجموعة من العناوين العامة تُوزَّع عند الحاجة وتُسترجع عند انتهاء الجلسة.

NAT — Advantages & Disadvantages مزايا وعيوب NAT

Advantages ✓

  • Conserves public IPv4 addresses dramatically.
  • Hides internal network topology (security benefit).
  • Allows internal IP renumbering without affecting Internet-facing address.
  • Enables multiple ISPs without internal renumbering.

Disadvantages ✗

  • Breaks end-to-end transparency (violates IP design philosophy).
  • Complicates protocols that embed IP addresses (FTP, SIP, H.323).
  • Adds latency and processing overhead at the router.
  • Makes inbound connections difficult (requires port forwarding).
Arabic — مزايا وعيوب
المزايا: يوفِّر عناوين IPv4 العامة، يُخفي البنية الداخلية للشبكة، يسمح بتغيير عناوين داخلية بدون تأثير على الخارج.
العيوب: يكسر مبدأ الشفافية الكاملة من طرف لطرف، يُعقِّد بعض البروتوكولات (مثل FTP و VoIP) التي تضمِّن عناوين IP في حمولتها، يضيف زمن معالجة على الموجِّه.

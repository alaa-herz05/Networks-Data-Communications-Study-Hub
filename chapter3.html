Application Layer — Overview

نظرة عامة على طبقة التطبيقات
What is the Application Layer? ما هي طبقة التطبيقات؟

The Application Layer is the topmost layer of the TCP/IP model. It provides the interface between the applications on either end of the network — the programs that users interact with daily (browsers, email clients, file managers) talk to the network through this layer.

  • It defines the rules and formats (protocols) that applications use to communicate.
  • It does NOT handle how data travels across the network — that is the job of lower layers.
  • Every application protocol operates at this layer: DNS, HTTP, FTP, SMTP, POP3, Telnet, DHCP.

FTP — File Transfer Protocol

بروتوكول نقل الملفات
FTP File Transfer Protocol Port 21 (Control) + Port 20 (Data)

FTP is one of the most commonly used application layer protocols. It allows for the transfer of files between clients and servers — both uploading (client → server) and downloading (server → client).

Arabic
FTP هو بروتوكول نقل الملفات — يُستخدم لرفع الملفات (Upload) وتنزيلها (Download) بين الكمبيوتر والسيرفر.
The Two Connections of FTP الاتصالان في FTP

FTP is unique among protocols because it requires TWO separate TCP connections simultaneously to function:

Connection 1 — Control
Port 21

Carries all commands and replies between client and server.
Examples: login, change directory, list files, quit.
This connection stays open the entire session.
Connection 2 — Data
Port 20

Carries the actual file content being transferred.
Opened for each file transfer, then closed.
Opened when a transfer is needed, then closed.

Arabic — لماذا اتصالان؟
FTP يحتاج اتصالين منفصلين لأن الأوامر والبيانات لهما طبيعة مختلفة جداً:
اتصال الأوامر (Port 21): يبقى مفتوحاً طوال الجلسة — يحمل الأوامر مثل تسجيل الدخول، تغيير المجلد، قائمة الملفات.
اتصال البيانات (Port 20): يُفتح فقط عند الحاجة لنقل ملف، ثم يُغلق بعد الانتهاء.
هذا الفصل يجعل التحكم أسرع واستخدام الموارد أكفأ.
FTP Exam Points نقاط الامتحان
  • FTP uses TCP (reliable, connection-oriented) for both connections.
  • Port 21 = commands/control. Port 20 = data transfer.
  • Supports both upload (client → server) and download (server → client).
  • FTP sends credentials (username/password) in plain text — NOT encrypted. (SFTP / FTPS are secure alternatives.)
  • FTP is considered a stateful protocol — the server remembers the client's state (current directory, etc.).
Common Exam Question
Q: Why does FTP need two connections?
A: Because commands/control (Port 21) and actual data (Port 20) serve different purposes and have different lifecycles. The control connection stays open throughout; the data connection opens and closes per transfer.

DHCP — Dynamic Host Configuration Protocol

بروتوكول الإعداد الديناميكي للمضيف
DHCP Dynamic Host Configuration Protocol Port 67 (Server) / Port 68 (Client)

DHCP enables devices to automatically obtain network configuration information from a DHCP server — eliminating the need to manually configure each device.

Arabic
DHCP يسمح للأجهزة بالحصول تلقائياً على معلومات الشبكة من السيرفر — بدل ما تضبط كل جهاز يدوياً.
What DHCP Assigns ماذا يُسنِد DHCP؟

When a device connects to a network, DHCP automatically provides:

  • ✅ IP Address — unique network identifier
  • ✅ Subnet Mask — defines the network boundary
  • ✅ Default Gateway — the router to reach other networks
  • ✅ DNS Server address — for name resolution
  • ✅ Other parameters: lease time, domain name, etc.
Real-World Example

When you connect to a café's Wi-Fi, DHCP automatically gives your phone an IP address, subnet mask, gateway, and DNS server — all within milliseconds. This is why you can use Wi-Fi at various places around the world!
Arabic
DHCP يُسنِد تلقائياً: عنوان IP + قناع الشبكة (Subnet Mask) + البوابة الافتراضية (Default Gateway) + عنوان سيرفر DNS
مثال: لما تتصل بـ Wi-Fi في المقهى، DHCP هو اللي يعطي جهازك IP تلقائياً خلال ثوانٍ.

الجهاز المتصل يطلب تجديد الاستئجار قبل انتهائه تلقائياً.
⚠️ When NOT to Use DHCP متى لا تستخدم DHCP؟

DHCP is great for regular users but MUST NOT be used for fixed infrastructure:

Do NOT use DHCP for:
  • 🖥️ Servers (web, email, file servers)
  • 🖨️ Printers
  • 🌐 Routers
  • 🔌 Switches
Why?
These devices need Static (permanent) IPs because clients must always find them at the same address. If a printer's IP changes overnight, no one can print in the morning!
Arabic
لا تستخدم DHCP للأجهزة الثابتة مثل السيرفرات والطابعات والراوترات.
السبب: هذه الأجهزة تحتاج عناوين IP ثابتة (Static) حتى يقدر المستخدمون والأجهزة الأخرى دائماً إيجادها على نفس العنوان.
مثال: لو IP الطابعة تغيّر بين ليلة وضحاها، ما أحد يقدر يطبع في الصبح!

Telnet — Remote Terminal Protocol

بروتوكول الطرفية عن بُعد
Telnet Remote Login Protocol Port 23

Telnet allows users to remotely log in to and control another computer over a network as if they were physically sitting at that machine.

Arabic
Telnet يتيح لك تسجيل الدخول والتحكم بجهاز آخر عن بُعد عبر الشبكة — كأنك جالس أمامه مباشرةً.
Telnet Key Facts معلومات مهمة عن Telnet
  • Developed in the early 1970s — one of the oldest application layer protocols in TCP/IP.
  • Allows users to emulate text-based terminal devices over the network using software.
  • A connection is called a virtual terminal (vty) session.
  • Can be run from the command prompt on a PC.
  • You get all the rights and priorities your username allows on the remote machine.
  • ⚠️ Security issue: Telnet sends all data (including passwords) in plain text — completely unencrypted. SSH (Port 22) is the modern secure replacement.
Telnet Use Case
Network engineers use it to remotely configure routers and switches. Type commands on your local computer, they execute on the remote device.
Security Warning ⚠️
Telnet transmits everything in plain text including usernames and passwords. Anyone on the network can intercept it. Use SSH (Port 22) instead in real environments.
Arabic
Telnet طُوِّر في السبعينيات وهو من أقدم بروتوكولات TCP/IP.
يُنشئ ما يُسمى بـ "جلسة طرفية افتراضية" (vty session) — تتحكم بالجهاز البعيد نصياً.
المشكلة الكبرى: كل شيء يُرسل بنص صريح (Plain Text) — بما في ذلك كلمة المرور! أي شخص على الشبكة يقدر يراها.
لهذا، في الأنظمة الحديثة نستخدم SSH (Port 22) بدل Telnet لأنه مشفَّر.

DNS — Name Space (Chapter 25.1 & 25.2)

نظام أسماء النطاقات — فضاء الأسماء
Why Do We Need DNS? لماذا نحتاج DNS؟

Computers communicate using IP addresses (e.g., 142.250.80.100). Humans communicate using names (e.g., google.com). DNS is the system that translates names to IP addresses automatically.

DNS = The Internet's Phone Book
Just as a phone book maps a person's name to their number, DNS maps a domain name to an IP address.
Arabic
الحواسيب تتواصل بعناوين IP (أرقام)، لكن البشر يتذكرون الأسماء بسهولة أكثر.
DNS هو النظام الذي يُترجم الاسم (مثل google.com) إلى عنوان IP تلقائياً كي تتصل بالسيرفر الصحيح.
بدون DNS سيضطر كل مستخدم لحفظ أرقام IP لكل موقع يزوره!

Arabic
فضاء الأسماء المسطّح (Flat): كل اسم فريد لكن لا يوجد تنظيم — مثل دليل هاتف واحد لكل العالم. غير قابل للتوسع.
فضاء الأسماء الهرمي (Hierarchical): تنظيم على شكل شجرة مقلوبة — الجذر (Root) في القمة، ثم النطاقات تتفرع للأسفل.
الإنترنت يستخدم الهرمي لأنه يتيح إدارة موزعة: كل منظمة تدير جزءها فقط.

توزيع فضاء الأسماء — سيرفرات DNS
Why Distribute DNS? لماذا نوزّع DNS؟

It would be very inefficient and unreliable to have just one computer store all DNS information for the entire Internet. Instead, DNS is distributed across a hierarchy of servers worldwide.

Arabic

الحل: توزيع DNS على آلاف السيرفرات حول العالم في هرمية منظَّمة.

Zone Transfer

نقل المنطقة
What is Zone Transfer? ما هو Zone Transfer؟
Zone Transfer is the process by which a Secondary DNS server downloads and synchronizes its DNS database from the Primary DNS server.

Key facts about zone transfer:

  • Primary server loads all information from a local disk file (zone file).
  • Secondary server loads all information from the Primary server via zone transfer.
  • Zone transfer ensures Secondary servers have up-to-date, consistent data.
  • Secondary servers can be notified of changes either actively (Primary pushes notification) or passively (Secondary periodically checks for updates).

Critical Exam Statement
"A primary server loads all information from the disk file; the secondary server loads all information from the primary server. When the secondary downloads information from the primary, it is called zone transfer."
— This exact statement is frequently quoted in exams.

DNS Name Resolution Process

عملية حل اسم النطاق
Step-by-Step: How DNS Resolves a Name خطوة بخطوة: كيف يحل DNS الاسم

When you type www.google.com in your browser:

1 Browser checks its local cache — if found, done!
2 Checks the hosts file on the local machine.
3 Sends query to the DNS Resolver (your ISP's or configured DNS server).
4 Resolver asks a Root Server: "Who handles .com?"
5 Root server replies with the address of the .com TLD server.
6 Resolver asks TLD server: "Who handles google.com?"
7 TLD server refers to Google's authoritative DNS server.
8 Google's DNS server returns the IP address for www.google.com.
9 Resolver returns IP to browser. Browser connects. Done!
Arabic — عملية الحل
لما تكتب www.google.com في المتصفح:
1- المتصفح يفحص Cache المحلي. 2- يفحص ملف hosts. 3- يسأل DNS Resolver (سيرفر DNS المُعيَّن لك).
4- الـ Resolver يسأل Root Server: "مَن يدير .com؟"
5- الـ Root يرد بعنوان سيرفر .com التابع لـ TLD.
6- الـ Resolver يسأل TLD: "مَن يدير google.com؟"
7- TLD يُحيله لسيرفر DNS الرسمي لـ Google.
8- سيرفر Google يرد بعنوان IP الخاص بـ www.google.com.
9- الـ Resolver يُرسل IP للمتصفح الذي يتصل بالسيرفر.

DDNS — Dynamic Domain Name System (Chapter 25.9)

نظام أسماء النطاقات الديناميكي
What is DDNS? ما هو DDNS؟

Traditional DNS uses static mappings stored in zone files — a human administrator must manually update them. But in modern networks (especially with DHCP), IP addresses change frequently. DDNS was devised to handle this automatically.

DDNS = DNS that updates itself dynamically when IP address bindings change.
Arabic
DNS التقليدي يستخدم mappings ثابتة في ملفات Zone يُعدّها المسؤول يدوياً.
لكن في شبكات DHCP، عناوين IP تتغيّر باستمرار — فكيف يعرف DNS الاسم الجديد للجهاز؟
DDNS حلّ هذه المشكلة بجعل DNS يُحدّث نفسه تلقائياً عند تغيُّر عناوين IP.

DNS Encapsulation — UDP vs TCP (Chapter 25.10)

تغليف DNS — متى UDP؟ متى TCP؟
DNS Transport: UDP or TCP? هل يستخدم DNS نقل UDP أم TCP؟

DNS can use either UDP or TCP. Both use the well-known port 53. The choice depends on the response size:

UDP — when response < 512 bytes

Most DNS queries are small and simple — a single domain name to IP mapping. UDP is faster because there is no connection setup overhead.
Most DNS queries use this path.
TCP — when response > 512 bytes

Zone transfers and large responses exceed the 512-byte UDP packet limit. TCP is used for reliability and to handle larger payloads.
Zone transfers always use TCP.
DNS Transport Decision Rule
Response size < 512 bytes → Use UDP (Port 53) — faster, no connection Response size > 512 bytes → Use TCP (Port 53) — reliable, handles large data Zone Transfer → Always TCP (Port 53)

DNS يعمل على Port 53 ويستخدم إما UDP أو TCP حسب حجم الرد:
UDP (أقل من 512 بايت): معظم استفسارات DNS صغيرة الحجم — UDP أسرع لأنه لا يحتاج إنشاء اتصال.
TCP (أكثر من 512 بايت): الردود الكبيرة وعمليات Zone Transfer تستخدم TCP لأنه أكثر موثوقية ويتعامل مع البيانات الكبيرة.
قاعدة مهمة للامتحان: Zone Transfer = TCP دائماً.
Key Exam Fact
DNS uses port 53 for BOTH UDP and TCP.
UDP for small queries (<512 bytes) | TCP for large responses (>512 bytes) and zone transfers.

Exam Questions & Answers

أسئلة الامتحان مع الإجابات
Concept Questions أسئلة مفاهيم
Q1
What is the role of the Application Layer in the TCP/IP model?
Answer: It provides the interface between the applications on either end of the network — defining the rules and formats (protocols) that user applications use to communicate over the network.
Q2
Why does FTP require two TCP connections? What does each carry?
Answer: FTP uses Port 21 for commands/control (login, directory commands, etc.) and Port 20 for actual data transfer (file content). They serve different purposes — the control connection stays open the entire session, while the data connection opens and closes per file transfer.
Q3
Why should servers, printers, and routers use static IPs instead of DHCP?
Answer: DHCP assigns temporary (leased) IP addresses that can change when the lease expires or the device reconnects. Servers and printers need permanent IPs so that clients can always reach them at the same address. A changing IP would break all connections to these devices.
Q4
What is a Zone Transfer? When does it occur?
Answer: Zone Transfer is when a Secondary DNS server downloads/synchronizes its DNS database from the Primary DNS server. It occurs when the secondary needs to update its data to match the primary's current zone file. It always uses TCP on Port 53.
Q5
What port does DNS use? When does it use UDP and when TCP?
Answer: DNS uses Port 53 for both protocols. UDP is used when the response is less than 512 bytes (most normal queries). TCP is used when the response exceeds 512 bytes or for Zone Transfers.
Q6
What is the difference between Flat and Hierarchical name spaces?
Answer: Flat: all names at the same level, each must be globally unique, not scalable. Hierarchical: inverted-tree structure with up to 128 levels, locally unique names within subtrees, distributed administration — DNS uses hierarchical.
Q7
What is DDNS and why was it created?
Answer: Dynamic DNS (DDNS) was created because IP addresses in DHCP networks change frequently. DDNS allows the DNS master file to be updated dynamically — when a DHCP server assigns a new IP, it notifies the Primary DNS server to update the zone, which then propagates to secondary servers.
Q8
What is a virtual terminal (vty) session and which protocol creates it?
Answer: A vty session is a virtual terminal session created by Telnet (Port 23). It allows a user to remotely log in to and control another computer as if physically present at that machine.
Q9
Name and describe 3 types of DNS servers.
Answer: (1) Root Server — top of hierarchy, knows addresses of all TLD servers. (2) Primary Server — authoritative source, loads data from local disk zone file. (3) Secondary Server — backup copy, downloads data from Primary via Zone Transfer.
Q10
List 5 application layer protocols with their port numbers.
Answer: DNS=53, HTTP=80, FTP=21/20, SMTP=25, Telnet=23, POP3=110, DHCP=67/68, HTTPS=443.
